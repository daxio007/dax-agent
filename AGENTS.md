# DAX Agent 协作约束

本文件是 Codex 和其他代码代理在本仓库中必须遵守的长期协作规则。

## 开发流程

1. 涉及新能力或重要行为变化时，先更新对应设计文档，再修改代码。
2. 所有 TypeScript 命名方法必须具有符合 `docs/jsdoc-standard.md` 的 JSDoc。
3. 每个方法参数都必须有清晰的 `@param` 用途说明。
4. 修改 TypeScript 源码后，必须运行：

```bash
npm run check:jsdoc
npm run typecheck
npm run build
```

5. 每次完成代码修改并通过必要验证后，必须把本次任务相关改动提交到当前 Git 分支，不能让已完成的代码停留在未提交状态。
6. Git 提交应包含本次代码、测试和相关文档，提交信息应准确概括改动。
7. 创建提交不等于推送远端；只有用户明确要求时才执行 `git push`。

## 修改边界

- 不回退或覆盖用户已有的无关修改。
- 提交前检查暂存内容，只提交当前任务相关文件。
- 如果验证失败，不得把任务描述为完成；应先修复，或明确记录无法完成的原因。
